Phase 1: specifying the format of your data

Data file: and after specifying the format below

The data is expected to be in a tabular format ASCII file in which columns are separated by some special separator character and each column contains data (some of which can be marked as missing) for one variable. (Alternatively, each row may contain data for each variable.) Different line ending encodings (Windows, MAC, Unix) are accepted. The first line of the data file may contain the variable names or the variable names can be automatically generated or uploaded in a separate file.

As a result of this phase the data is converted to the tab-limited format, one variable per (newline terminated) row in which missing values are marked as empty strings and the data content is analysed to form the basis for the next phase, the discretization.

Variable processing command lines

For leaving variables out of analysis and for specifying separate missing marker for different variables.
Command line syntax:
cmdline::= ranges :: cmd
ranges::= range (, range)*
range::= var | [var] - [var]
var::= variable name | variable index (from 0 on)
cmd::= in | out | miss mark*
